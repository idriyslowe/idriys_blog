"study-app"

My first product. Learning to build a RoR app mostly on my own

Github Link ==>HERE

Nevermind the inappropriate hyphen-named repo. I was still learning naming conventions.

My bootcamp's final project was to make an app of our own designing using whatever tickled our fancies. I chose to being with RoR and it expanded from there.

Origin Story: Before bootcamp, I tried to study many programming languages and frameworks before having a full grasp of the context of anything I was learning. My notes very quickly went from legible to rat's nest. I didn't plan for all of the side notes and bookmarks. I needed a place to organize my "Look up this term" and "What's the diff between these methods?" notes. Now, I'm not an idiot. I know things like Evernote exist but it was the only thing I could think of... I just learned how to do all of this a second ago!

I began with a spreadsheet that outlined my tables. Environment: This is the broadest subject. Think "Programming". Notepage: An Environment has many of these but each Notepage belongs to an Environment. This is where you'd break "Programming" down to "Rails". Note: A Notepage has many of these (listed on it's show page). Each one belongs to a Notepage. This are your short notes like you would write in a notebook. Bookmark: These also belong to a Notepage to start but can be changed to belong to an individual note. A Bookmark object is comprised of a screenshot from the web and a url so the user can see the live page. Comment: Allows users to comment on their own or others' Notepages. This appears on the Notepage. Obviously, I need a User table (I chose the Devise gem to handle the password encryption).

The second step was building the Rails site real ugly-like. I generated my models and controllers quickly knowing I'd learn what I was missing in the process. I hand wrote CRUD routes for each resource. I wasn't sure yet what my app would look like at the time so I started building the views in the order that a user would discover them starting with the Environment index, going through Environnment show (listing its Notepages) and terminating at the Notepage show view.

The Notepage show was baffling for me. I had many ideas about what I wanted the user to be able to do here but they were all too ambitious for the time contraint. Finally, it lists the Notes, Bookmarks, and Comments. One page to rule them all! RoR wasn't going to cause me trouble here. I thought of the user who will be providing lots on input. A user won't want to click a button and wait for a refresh for every edit. It was time for Angular.

I downloaded the angular.min file and created an Angular NotePage controller to do take over the Note iterations and updates in the view. Using Angular meant that this app needed to communicate... it needed to output data for my Angular controller to consume. Now it's time for JSON.

My app had to grow from CRUD app to JSON REST API. First try, Jbuilder. Deeply nested JSON with Jbuilder was a terrible and time-consuming failure. I needed to list the Bookmarks that belonged to the Notes that belonged to the Notepage that we were on... all of those things at once. Rabl came to the rescue! Created an API controller folder for namespacing sake. My routes file grew like a weed and were eventually trimmed. Some actions were only called on the API controllers and some on the normal controllers.
